# Ground Truth(정답 텍스트) 작성 가이드

> 목적: 1-3 Whisper 오류 분포 측정을 위한 정답 텍스트(GT) 작성 시 상황별 처리 기준 정리
> 관련 계획: [voice_intent_classification_plan.md](voice_intent_classification_plan.md) § 1-2, 1-3

---

## 핵심 원칙

GT의 목적은 **Whisper가 무엇을 잘못 인식하는지 측정**하는 것이다.
따라서 정답은 화자가 실제로 발화한 내용 그대로여야 한다.

- 안 들린다고 임의로 채우거나 생략하면 → 오류 분포가 왜곡됨
- 깔끔하게 "정제"하고 싶은 충동을 억제할 것 → 측정값이 실제보다 낮게 나옴

---

## 상황별 처리 기준

### 1. 일부 단어가 불명확하게 들릴 때 (`partial`)

발화 속도, 배경 소음, 발음 불량 등으로 단어가 뭉개지는 경우.

**처리 방법**
1. 해당 구간을 여러 번 반복 청취
2. 최선을 다해 추정한 내용을 그대로 기록
3. 추정이 불확실하면 `[unclear: 추정단어]` 형식으로 표기
4. 완전히 추정 불가능한 단어는 `[inaudible]`로 표기

**예시**
```
"저한테 [unclear: 대출] 신청해 주시면 바로 처리해 드리겠습니다"
"[inaudible] 계좌로 입금해 주시면 됩니다"
```

**측정 처리**: 포함 (`quality: partial`)

---

### 2. 발화 없이 묵음·배경음만 있는 구간

통화 연결음, 잡음, 침묵 구간.

**처리 방법**
- 해당 구간은 전사하지 않음 (공백/빈 문자열)
- Whisper가 이 구간에 환각(hallucination) 텍스트를 생성하면 → **insertion 오류**로 자동 계산됨

**측정 처리**: 포함 (`quality: good`) — Whisper 환각 패턴 측정에 중요한 샘플

---

### 3. 소음에 완전히 묻혀 아무것도 안 들릴 때 (`bad`)

절반 이상의 발화가 식별 불가능한 경우.

**처리 방법**
- 해당 파일을 측정 샘플에서 **제외**
- 이유: 극단적인 음질의 샘플이 섞이면 오류 분포 측정값이 튀어서 실제 서비스 환경과 달라짐

**측정 처리**: 제외 (`quality: bad`)

> 💡 `bad` 샘플을 제외해도 수렴 기준(클래스별 최소 30개)은 나머지로 충분히 달성 가능하다.
> 오히려 깨끗한 샘플로 측정해야 ASR 오류 분포가 더 정확하게 나온다.

## CSV 작성 형식

[sample_ground_truth.py](../../models/classifier/preprocessing/sample_ground_truth.py)가 생성하는 CSV에 아래 컬럼을 사용한다.

| 컬럼 | 설명 | 값 예시 |
|------|------|---------|
| `filename` | 음성 파일명 | `001.mp3` |
| `whisper_text` | Whisper 전사 결과 (자동 생성) | `저금리 대출 받으시려면` |
| `ground_truth` | 사람이 직접 작성한 정답 텍스트 | `저금리 대출 받으시려면` |
| `quality` | 음질 품질 등급 | `good` / `partial` / `bad` |
| `label` | 의도 레이블 | `phishing` / `normal` |
| `category` | 피싱 유형 또는 일반 구분 | `대출 사기형` |
| `notes` | 특이사항 메모 (선택) | `[unclear: 대출] 구간 있음` |

**작성 예시**

| filename | whisper_text | ground_truth | quality | label | category | notes |
|---|---|---|---|---|---|---|
| 001.mp3 | 저금리 대출 받으시려면 먼저 수수료를 | 저금리 대출 받으시려면 먼저 수수료를 | good | phishing | 대출 사기형 | |
| 002.mp3 | 계좌 번호 알려주시면 바로 | [unclear: 계좌] 번호 알려주시면 바로 | partial | phishing | 대출 사기형 | 첫 단어 불명확 |
| 003.mp3 | (Whisper 환각 텍스트) | | good | phishing | 대출 사기형 | 묵음 구간, Whisper 환각 확인용 |
| 004.mp3 | (제외) | (제외) | bad | phishing | 대출 사기형 | 전체 구간 소음으로 식별 불가 |

## 샘플 수 및 클래스 기준

[voice_intent_classification_plan.md § 1-3](voice_intent_classification_plan.md) 기준 요약:

| 항목 | 기준 |
|---|---|
| 최소 샘플 수 | 클래스별 30개 이상 (총 60개~) |
| 목표 샘플 수 | 200~300개 |
| 클래스 비율 | 보유 데이터 자연 비율 유지 (균형 강제 불필요) |
| 수렴 기준 | 연속 3구간(30개) 동안 오류율 변화 ±0.01 이하 |
| `bad` 샘플 | 측정에서 제외, 별도 집계만 유지 |

---

## 주의사항

1. **`partial` 샘플은 포함한다** — `[unclear]` 표기가 있어도 나머지 구간의 오류는 정상 측정됨
2. **정제 충동을 억제한다** — 실제 발화대로 적어야 Whisper 오류가 올바르게 측정됨
3. **`bad` 샘플은 별도 집계한다** — 전체 데이터 중 몇 %가 사용 불가인지 파악해두면 데이터 수집 계획에 활용 가능
4. **CPU/GPU 버전 각각 별도로 작성하지 않아도 된다** — GT는 하나이며, Whisper 전사 결과(`whisper_text`)만 버전별로 다름
